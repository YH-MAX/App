4. Backend Implementation (Spring Boot)

4.1 Overview

The SmartWater Backend is a RESTful API service built with Spring Boot 3.5.7, serving as the core server infrastructure for the mobile application. It provides user authentication, real-time water quality monitoring, community engagement features, and pollution reporting capabilities.

4.2 Technology Stack

Technology Version Purpose
Spring Boot 3.5.7 Core REST API framework
Spring Security + JWT 6.x / 0.11.5 Authentication and authorization
Spring Data JPA 3.x Database ORM (Object-Relational Mapping)
MySQL 8.0 Relational database storage
Spring Mail - Email verification and password reset
Spring WebFlux - Reactive HTTP client for FastAPI integration
Java 17 Programming language
Gradle 8.x Build automation tool

4.3 System Architecture

The backend follows a three-tier layered architecture pattern with clear separation of concerns:

  • Presentation Layer (Controllers): 8 REST controllers handling HTTP requests and responses
  • Business Logic Layer (Services): 12 service classes containing core application logic
  • Data Access Layer (Repositories): 13 JPA repository interfaces for database operations

The system integrates with an external FastAPI IoT service (port 8888) for real-time sensor data. The data flow is illustrated below:

Backend ↔ FastAPI Integration Data Flow

[Mobile App] → [Spring Boot :8080] → [WebClient] → [FastAPI :8888] → [Sensor JSON]
                                    ↓
[Mobile App] ← [Spring Boot :8080] ← [Process Data] ← [Return Response] ←─────────┘

4.6 Database Design

The MySQL database consists of 13 tables managed by Spring Data JPA with Hibernate:

Category Tables Description
User Management users, email_verification_tokens, password_reset_tokens User accounts, email verification, password recovery
Community community_posts, community_replies, post_likes, post_bookmarks, post_retweets, user_follows Social platform features
Monitoring alerts, sensor_data, pollution_reports, bluetooth_devices Water quality monitoring and IoT devices

Key Entity Relationships:

  • User → Posts/Alerts/Devices: One-to-Many relationships
  • User → User (Follows): Many-to-Many via user_follows junction table
  • Post → Replies/Likes/Bookmarks: One-to-Many relationships

4.7 Security Implementation

The backend implements comprehensive security measures:

  • JWT Authentication: Stateless token-based authentication with HMAC-SHA256 signing and 24-hour expiration
  • Password Security: BCrypt hashing algorithm with random salt
  • Email Verification: Required before account activation; verification link sent via Gmail SMTP
  • Protected Endpoints: All sensitive endpoints require valid JWT token in Authorization header
  • CORS Configuration: Cross-origin requests enabled for mobile app integration
