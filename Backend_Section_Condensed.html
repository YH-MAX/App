3.4.2 Backend Database Schema (MySQL)

The Spring Boot backend uses MySQL 8.0 with 13 relational tables managed by Spring Data JPA:

Category Tables Purpose
User Management users, email_verification_tokens, password_reset_tokens Authentication, registration, password recovery
Community community_posts, community_replies, post_likes, post_bookmarks, post_retweets, user_follows Social features: posts, likes, follows
Monitoring alerts, sensor_data, pollution_reports, bluetooth_devices Water quality alerts, sensor data, IoT devices

Key Relationships:

  • User → Posts/Alerts/Devices (One-to-Many)
  • User → User (Many-to-Many via user_follows)
  • Post → Replies/Likes/Bookmarks (One-to-Many)

4.2 Backend Implementation (Spring Boot)

4.2.1 Technology Stack

Technology Version Purpose
Spring Boot 3.5.7 Core REST API framework
Spring Security + JWT 6.x / 0.11.5 Authentication & authorization
Spring Data JPA 3.x Database ORM
MySQL 8.0 Relational database
Spring Mail - Email verification
WebFlux/WebClient - FastAPI integration

4.2.2 Architecture

The backend follows a three-tier layered architecture:

  • Presentation Layer: 8 REST Controllers handling HTTP requests
  • Service Layer: 12 Service classes containing business logic
  • Data Access Layer: 13 JPA Repositories for database operations

4.2.3 Core Modules

Module Components Functionality
User Authentication UserController, UserService, JwtUtil Registration, login (JWT), email verification, password reset
Sensor Integration SensorController, FastApiSensorFacadeService Proxy to FastAPI IoT service for real-time water data
Alert System AlertController, AlertService Water quality threshold monitoring (pH, temperature, turbidity)
Community Platform CommunityController, CommunityService Posts, likes, bookmarks, replies, search
Pollution Reporting PollutionReportController, PollutionReportService Report submission, status tracking (PENDING→RESOLVED)
Bluetooth/IoT BluetoothController, BluetoothService Device pairing and connection status

4.2.4 API Endpoints Summary

The backend exposes 40+ RESTful API endpoints across 7 endpoint groups:

Endpoint Group Base Path Key Operations
Authentication /api/users register, login, verify-email, reset-password, profile
Sensor Data /api/sensor latest readings, historical range, summary
Alerts /api/alerts evaluate water quality, alert history
Community /api/community posts CRUD, like, bookmark, retweet, reply, search
Reports /api/reports create report, status update, list reports
Social /api/users/{id} follow/unfollow, followers list, profile
Bluetooth /api/bluetooth device pairing, connection status

4.2.5 Security Implementation

  • Authentication: JWT tokens with 24-hour expiration, HMAC-SHA256 signing
  • Password Security: BCrypt hashing with salt
  • Email Verification: Required before login activation
  • Protected Endpoints: Authorization header with Bearer token required

4.2.6 External Integration

The backend integrates with:

  • FastAPI IoT Service (Port 8888): Real-time sensor data from water monitoring devices
  • Gmail SMTP: Email verification and password reset functionality
