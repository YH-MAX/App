1. System Overview

The SmartWater Monitoring System is a comprehensive IoT-based platform designed to monitor and analyze water quality in real-time. The system employs a three-tier microservices architecture that separates concerns across presentation, business logic, and data layers.

Key Objectives:
  • Real-time water quality monitoring (pH, Temperature, Turbidity)
  • Automated alert generation for threshold violations
  • Community engagement platform for water quality reporting
  • IoT device management via Bluetooth connectivity
  • Pollution incident reporting and tracking

3. Three-Tier Architecture

The system follows a classic three-tier architecture pattern with clear separation of concerns:

📱 Presentation Layer (Android App)

  • Technology: Native Android (Java)
  • Pattern: Activity-based architecture with Retrofit networking
  • Responsibilities:
    • User interface rendering and interaction
    • Local data caching and state management
    • Bluetooth device connectivity
    • Real-time data visualization with charts

⚙️ Business Logic Layer (Spring Boot)

  • Technology: Spring Boot 3.5.7 (Java 17)
  • Pattern: Layered architecture (Controller → Service → Repository)
  • Responsibilities:
    • RESTful API endpoints
    • Business rule validation and processing
    • JWT-based authentication and authorization
    • Integration with IoT data service
    • Alert generation and notification

🗄️ Data Layer (MySQL + SQLite)

  • Primary Database: MySQL 8.0 (User data, community, alerts)
  • IoT Database: SQLite (Time-series sensor data)
  • Pattern: JPA/Hibernate ORM
  • Responsibilities:
    • Persistent data storage
    • ACID transaction support
    • Relational data integrity

  • CommunityService: Posts, likes, bookmarks, retweets, replies

  • FollowService: Social graph management
  • PollutionReportService: Report lifecycle management
  • BluetoothService: Device pairing and status tracking
  • EmailService: SMTP email sending
  • EmailVerificationService: Email token management
  • PasswordResetService: Password reset flow
  • FastApiSensorFacadeService: WebClient calls to FastAPI
  • WaterProxyService: RestTemplate proxy to FastAPI
  • CustomUserDetailsService: Spring Security integration

JPA repositories for all entity types with custom query methods for pagination, filtering, and relationship traversal.

8. API Architecture

RESTful API Design Principles

  • Resource-based URLs: /api/users, /api/community/posts, /api/alerts
  • HTTP Methods: GET (read), POST (create), PUT (update), DELETE (remove)
  • JSON Response: Consistent JSON response format with proper status codes
  • Pagination: PageResponse DTO for paginated results
  • Error Handling: GlobalExceptionHandler with ApiError response format

API Endpoint Summary

Module Endpoints Description

Sensor Data 5 endpoints Device readings, time-range queries, data upload
