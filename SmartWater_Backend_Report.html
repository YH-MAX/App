1. Introduction

The SmartWater Backend is a robust RESTful API service developed using the Spring Boot 3.5.7 framework, serving as the core backend infrastructure for the SmartWater Monitoring System. This backend powers the Android mobile application, providing essential functionalities including user authentication, real-time water quality monitoring, community engagement, and pollution reporting.

The backend follows a layered architecture pattern with clear separation of concerns, implementing industry-standard security practices including JWT-based authentication and BCrypt password encryption.

1.1 Project Objectives

  • Provide secure user authentication and authorization
  • Enable real-time water quality data monitoring
  • Implement community engagement features for information sharing
  • Support pollution incident reporting and tracking
  • Integrate with IoT sensor devices via FastAPI service

5. Key Features Implementation

5.1 User Authentication & Management

The authentication system implements JWT (JSON Web Token) based stateless authentication:

  • Registration: Users register with email, username, and password. Passwords are hashed using BCrypt algorithm.
  • Email Verification: A verification token is sent to the user's email. Account activation requires email confirmation.
  • Login: Upon successful authentication, a JWT token is generated with 24-hour expiration.
  • Password Reset: Users can request password reset via email with a secure token.
  • Profile Management: Users can update their profile information including avatar, bio, and location.
Security Flow:
Register → Email Verification → Login → JWT Token → Access Protected APIs

5.2 Sensor Data Integration

The backend integrates with an external FastAPI IoT service to retrieve real-time water quality sensor data:

  • Latest Reading: Fetch the most recent sensor reading for a specific device
  • Historical Data: Query sensor data within a specified time range
  • Summary Statistics: Generate water quality summary including averages and status

Integration Method: Spring WebClient (reactive, non-blocking HTTP client)

5.3 Alert System

The alert system monitors water quality parameters and generates alerts based on predefined thresholds:

Parameter Critical Threshold Warning Threshold
pH Level < 6.0 or > 8.5 6.0-6.5 or 8.0-8.5
Temperature > 40°C > 30°C
Turbidity > 100 NTU > 50 NTU

Alert Severity Levels:

  • HIGH: Immediate action required (critical thresholds exceeded)
  • MEDIUM: Attention needed (warning thresholds exceeded)
  • LOW: Informational alerts

11. Code Implementation Examples

11.1 JWT Token Generation (JwtUtil.java)

@Component
public class JwtUtil {

    @Value("${jwt.secret}")
    private String secretKey;

    @Value("${jwt.expiration}")
    private Long expirationMs;

    private Key getSigningKey() {
        return Keys.hmacShaKeyFor(secretKey.getBytes());
    }

    public String generateToken(String username) {
        return Jwts.builder()
                .setSubject(username)
                .setIssuedAt(new Date())
                .setExpiration(new Date(System.currentTimeMillis() + expirationMs))
                .signWith(getSigningKey(), SignatureAlgorithm.HS256)
                .compact();
    }

    public boolean isTokenValid(String token, UserDetails userDetails) {
        final String username = extractUsername(token);
        return (username.equals(userDetails.getUsername())) && !isTokenExpired(token);
    }
}

11.2 User Login Endpoint (UserController.java)

@PostMapping("/login")
public ResponseEntity<Map<String, Object>> login(@RequestBody Map<String, String> loginRequest) {
    String email = loginRequest.get("email");
    String password = loginRequest.get("password");

    // Authenticate user
    authenticationManager.authenticate(
        new UsernamePasswordAuthenticationToken(email, password)
    );

    // Check if email is verified
    User user = userService.findByEmail(email);
    if (!user.isEmailVerified()) {
        return ResponseEntity.status(HttpStatus.FORBIDDEN)
            .body(Map.of("error", "Email not verified"));
    }

    // Generate JWT token
    String token = jwtUtil.generateToken(email);

    Map<String, Object> response = new HashMap<>();
    response.put("token", token);
    response.put("user", user);

    return ResponseEntity.ok(response);
}

11.3 Alert Evaluation Service (AlertService.java)

@Service
public class AlertService {

    public AlertResponse evaluateWaterQuality(WaterReadingRequest reading, User user) {
        String message = "";
        String severity = "LOW";

        // Check pH levels
        if (reading.getPh() < 6.0 || reading.getPh() > 8.5) {
            message = "Critical: pH level out of safe range!";
            severity = "HIGH";
        } else if (reading.getPh() < 6.5 || reading.getPh() > 8.0) {
            message = "Warning: pH level approaching unsafe range";
            severity = "MEDIUM";
        }

        // Check temperature
        if (reading.getTemperature() > 40) {
            severity = "HIGH";
            message += " Critical temperature detected!";
        } else if (reading.getTemperature() > 30) {
            if (!severity.equals("HIGH")) severity = "MEDIUM";
            message += " Elevated temperature warning.";
        }

        // Save alert to database
        Alert alert = new Alert();
        alert.setUser(user);
        alert.setMessage(message);
        alert.setSeverity(severity);
        alertRepository.save(alert);

        return new AlertResponse(severity, message);
    }
}
