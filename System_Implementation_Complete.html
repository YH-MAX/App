4. System Implementation

This section describes the implementation of the SmartWater Monitoring System, covering both the Android frontend application and the Spring Boot backend server.

System Architecture

                             ↓

4.1 Frontend Implementation (Android)

4.1.1 Technology Stack

Technology Purpose
Java Main programming language
Android SDK (API 24+) Android development framework
Retrofit 2 HTTP client for API calls
MPAndroidChart Charts for water quality data visualization
SharedPreferences Local storage for JWT token
Google Maps API Location picking for reports

4.1.2 App Structure

The Android app consists of 18 Activity screens and a network layer for API communication:

Module Activities Function
Authentication LoginActivity, RegisterActivity User login and registration with email verification
Dashboard DashboardActivity Display real-time water quality data with charts, auto-refresh every 5 seconds
Alerts AlertActivity Show water quality alerts sorted by severity (HIGH/MEDIUM/LOW)
Community CommunityActivity, CreatePostActivity, PostDetailActivity, SearchActivity Social feed with posts, likes, bookmarks, replies
Reports ReportActivity, SubmitReportActivity, MapPickerActivity Submit and view pollution reports with location
Profile ProfileActivity, UpdateProfileActivity, UserProfileActivity, FollowListActivity User profile management and social following
Bluetooth BluetoothActivity Pair and manage IoT sensor devices

4.1.3 Network Layer

The app uses Retrofit to communicate with the backend. The network layer includes:

  • ApiClient: Configures Retrofit with base URL and JWT interceptor
  • JwtInterceptor: Automatically adds Authorization header to requests
  • TokenStore: Manages JWT token storage in SharedPreferences

API interfaces are organized by feature:

API Interface Endpoints
AuthApi login, register, verify-email
WaterApi sensor data, summary, history
AlertApi evaluate, get alerts
CommunityApi posts, likes, bookmarks, replies
ReportApi create/view pollution reports
BluetoothApi device pairing, status

4.1.4 Key Features

  • Real-time Dashboard: Water quality data (pH, temperature, turbidity) with line charts that update every 5 seconds
  • Alert System: Color-coded alerts (red for HIGH, orange for MEDIUM, yellow for LOW)
  • Community Feed: Twitter-style posts with like, retweet, bookmark, and reply functions
  • Location-based Reports: Uses Google Maps to pick exact location when submitting pollution reports
  • Bluetooth Device Pairing: Connect to IoT water sensors and track connection status

4.2 Backend Implementation (Spring Boot)

4.2.1 Technology Stack

Technology Version Purpose
Spring Boot 3.5.7 REST API framework
Spring Security + JWT 6.x Authentication
Spring Data JPA 3.x Database ORM
MySQL 8.0 Database
Spring WebFlux - FastAPI integration

4.2.2 Architecture

The backend uses a three-layer architecture:

  • Controllers (8): Handle HTTP requests (UserController, SensorController, CommunityController, etc.)
  • Services (12): Business logic (AlertService, CommunityService, EmailService, etc.)
  • Repositories (13): Database operations using Spring Data JPA

4.2.3 Core Modules

Module Function
User Authentication JWT login, registration, email verification, password reset
Sensor Data Proxy to FastAPI for real-time water quality readings
Water Gateway Direct FastAPI proxy endpoints (/api/water/latest, /api/water/history)
Alert System Evaluate water readings against thresholds (pH 6.5-8.5, turbidity < 5 NTU)
Community Posts, likes, bookmarks, retweets, replies, search
Reports Pollution report submission with status tracking
Bluetooth IoT device pairing and connection management

4.2.4 API Endpoints

The backend provides 45+ REST endpoints organized into 8 groups:

Group Base Path Key Operations
Auth /api/users register, login, verify-email, reset-password
Sensor /api/sensor device/{id}/latest, me/latest, me/range, me/summary
Water Gateway /api/water latest, history, test-connection
Alerts /api/alerts evaluate, me (history)
Community /api/community posts, like, bookmark, retweet, replies, search
Reports /api/reports create, list, update status
Bluetooth /api/bluetooth pair, status, devices

4.2.5 Database

MySQL database with 13 tables:

  • User management: users, email_verification_tokens, password_reset_tokens
  • Community: community_posts, community_replies, post_likes, post_bookmarks, post_retweets, user_follows
  • Monitoring: alerts, sensor_data, pollution_reports, bluetooth_devices

4.2.6 Security

  • JWT tokens with 24-hour expiration
  • Passwords hashed with BCrypt
  • Email verification required before login
  • Protected endpoints require Authorization header
