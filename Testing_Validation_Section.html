5. Testing And Validation

5.1 Backend API Testing

We used IntelliJ's built-in HTTP Client to test all our backend APIs. For each API endpoint, we checked if the request returns the correct response and the data is stored properly in the database.

Here are some of the main tests we ran:

API What We Tested Result
Login User enters email and password, server returns JWT token Works
Get Sensor Data Fetch latest water quality reading from device WATER_001 Works
Alert Evaluation Submit water readings with low pH (5.2), check if alert is generated Works - returns HIGH severity
Create Post Post new message to community feed Works
Get User Profile Retrieve current user's profile info Works

5.2 Security Testing

We tested the JWT authentication to make sure protected APIs cannot be accessed without login:

  • When we tried to access /api/alerts/me without a token, the server returned 403 Forbidden - this is correct
  • When we used an expired token, the server returned 401 Unauthorized
  • When we used a valid token, the server returned the expected data

We also checked the database directly and confirmed that passwords are stored as BCrypt hashes, not plain text.

5.3 End-to-End Testing

We tested the complete flow from the Android app through the backend to the database to make sure everything works together:

Test 1: User Registration and Login Flow

  1. Open the app and tap "Register"
  2. Enter email, password, and username
  3. Check email inbox for verification link
  4. Click the link to verify account
  5. Go back to app and login with the registered credentials
  6. Result: User successfully logged in and redirected to Dashboard

Test 2: Water Quality Monitoring Flow

  1. Login to the app
  2. View Dashboard - check if sensor data is displayed
  3. Data refreshes every 5 seconds automatically
  4. When pH goes below 6.5, alert notification appears
  5. Result: Real-time data flows correctly from FastAPI → Backend → Android App

Test 3: Community Post Flow

  1. Go to Community tab
  2. Create a new post about water quality
  3. Check if the post appears in the feed
  4. Like and bookmark the post
  5. Result: Post saved to database and displayed correctly

Test 4: Pollution Report Flow

  1. Go to Reports section
  2. Submit a new pollution report with location
  3. Check the report appears in "My Reports"
  4. Result: Report saved with OPEN status

5.4 Issues Found and Fixed

During testing, we encountered a few issues:

  • Initially the Community and Reports APIs returned 403 error even for GET requests. We fixed this by updating SecurityConfig.java to allow public GET access
  • The Android app was trying to connect to localhost, which doesn't work on a real phone. We changed it to use the actual server IP address
  • Dashboard data wasn't updating because FastAPI service wasn't running. We need to start both backend server and FastAPI before using the app

5.5 Conclusion

All the main features have been tested from end to end. The system works correctly when all components (Android App, Spring Boot Backend, FastAPI Service, MySQL Database) are running together.
